## 快速开始

### 1. 创建你的第一个目标

进入 **Goals** 页面，点击「New Goal」，设定一个长期或短期目标。目标是你所有工作的方向指引。

### 2. 创建项目

在 **Projects** 页面创建一个项目，描述它的交付物（deliverable），并关联到目标。

### 3. 添加任务

在 **Tasks** 页面或项目详情中添加任务。为任务设置优先级（P1/P2/P3）和计划日期（planDate），让它出现在今天的任务列表中。

### 4. 开始专注

回到 **Dashboard**，在 FocusZone 中选择一个任务，点击「Start」开始你的第一个番茄钟。

## 核心概念

### 领域层级

VibeFlow 采用四层结构来组织你的工作：

```
目标 (Goals)       — 1 周到 5 年的愿景
  └── 项目 (Projects)  — 任务的容器，有明确交付物
        └── 任务 (Tasks)   — 具体的待办事项（P1/P2/P3 优先级）
              └── 番茄钟 (Pomodoros) — 10–120 分钟的专注会话
```

### 系统状态

系统有三种核心状态，决定了你在每个时刻能做什么：

| 状态 | 含义 | 可执行操作 |
|------|------|-----------|
| **IDLE（空闲）** | 准备好开始新的专注 | 开始番茄钟、管理任务和项目 |
| **FOCUS（专注）** | 正在进行深度工作 | 完成/中止番茄钟、切换任务 |
| **OVER_REST（超时休息）** | 休息时间过长 | 开始新番茄钟、等待冷却后返回空闲 |

**状态流转**：

```
IDLE  ──开始番茄钟──>  FOCUS
FOCUS ──完成番茄钟──>  IDLE ──休息过长──>  OVER_REST
FOCUS ──中止番茄钟──>  IDLE（不触发超时休息）
OVER_REST ──开始番茄钟──>  FOCUS
OVER_REST ──冷却 10 分钟后──>  IDLE
```

### 每日上限

默认每天 8 个番茄钟。达到上限后无法再开始新的番茄钟。可在设置中调整。

### 每日重置

每天凌晨 4:00 自动重置，清空当日番茄计数、状态回到 IDLE。

## Dashboard（仪表盘）

Dashboard 是你的指挥中心，包含以下区域：

### FocusZone（专注区）

位于页面顶部，根据当前状态显示不同内容：

- **IDLE 状态**：显示任务选择器 + 「开始」按钮 + 番茄进度点（●●●○○○○○ 3/8）
- **FOCUS 状态**：显示当前任务名 + 倒计时（MM:SS）+ 进度条 + 中止按钮
- **完成时**：弹出完成对话框，选择评价并进入休息
- **休息中**：休息倒计时 + 结束休息按钮

### 今日习惯

显示今天需要完成的习惯打卡状态。

### 今日任务列表

显示今天计划的所有任务，支持内联操作（完成、编辑等）。点击任务可打开详情面板。

### 辅助区域

- **Daily Progress（每日进度）**：番茄完成情况、工作时间统计
- **Focus Session（专注会话）**：工作时段外的干扰屏蔽控制

### Suggestions（建议）

可折叠区域，包含：
- **目标风险建议**：如果某个目标进度落后，给出提醒
- **任务建议**：基于优先级和剩余时间的智能任务推荐

## 任务管理

### 创建任务

1. 点击 Tasks 页面的「New Task」按钮
2. 填写任务标题、描述
3. 选择所属项目
4. 设置优先级（P1 最高 / P2 中等 / P3 最低）
5. 设置计划日期（planDate）— 只有设了今天日期的任务才会出现在"今日任务"中

### 任务筛选

Tasks 页面提供三个筛选视图：

| 视图 | 说明 |
|------|------|
| **Today** | 计划日期为今天的任务 |
| **Overdue** | 过期未完成的任务（红色高亮） |
| **Backlog** | 没有设置计划日期的任务 |

### 任务层级

任务支持子任务（subtasks），可以逐级分解复杂任务。在任务详情中可以添加子任务。

### 按项目查看

页面下方按项目分组展示任务，方便查看各项目的任务分布。

### 专注中的任务

如果当前有番茄钟在运行，任务页面顶部会显示当前专注会话的倒计时和任务名称。

1. 进入 Projects 页面，点击「New Project」
2. 输入项目标题和交付物描述
3. 可选：关联到一个或多个目标

### 项目状态

| 状态 | 含义 |
|------|------|
| **Active** | 进行中的项目 |
| **Completed** | 已完成的项目 |
| **Archived** | 已归档的项目 |

### 项目进度

如果项目中的任务设置了时间估算，项目卡片会显示：
- 完成进度百分比
- 已完成番茄数 / 总预估番茄数
- 进度条（超出预估时变红）

### 关联目标

项目可以关联到目标，卡片底部会显示紫色的目标标签。

## 番茄钟（专注会话）

### 开始番茄钟

在 Dashboard 的 FocusZone 中：

1. 从下拉列表选择要专注的任务（或选择「无任务」模式）
2. 点击「Start」按钮
3. 系统进入 FOCUS 状态，倒计时开始

### 专注中

- 页面显示倒计时和进度条
- 可以**切换任务**：如果中途需要处理另一个任务
- 可以**完成当前任务**：标记当前任务完成后继续番茄钟
- 可以**中止**：放弃本次番茄钟（不计入完成数，不触发超时休息）

### 完成番茄钟

倒计时结束后：
1. 弹出完成对话框
2. 可选择评价本次专注质量
3. 番茄计数 +1
4. 进入休息阶段

### 多任务支持

一个番茄钟内可以切换多个任务，系统会自动记录每个任务的时间分片（Time Slice），便于后续统计各任务的实际投入时间。

### 无任务番茄钟

如果你想先开始工作、后续再关联任务，可以启动「无任务番茄钟」（Taskless Pomodoro）。

### 休息与超时休息

完成番茄钟后进入休息。如果休息时间过长，系统会自动进入 **OVER_REST** 状态：

- OVER_REST 状态下，浏览器扩展和桌面端会启动屏蔽模式
- 需要等待 10 分钟冷却期后才能手动返回 IDLE
- 每天最多 3 次手动返回 IDLE
- 也可以直接开始新的番茄钟来退出 OVER_REST
- 工作时间结束时自动退出 OVER_REST

## 习惯追踪

### 创建习惯

在 Habits 页面点击「创建习惯」：

1. 输入习惯名称
2. 选择类型：

4. 可选：设置提醒时间

### 每日打卡

Dashboard 上的「Today's Habits」区域显示今天需要完成的习惯，可直接打卡。

### 管理习惯

在 Habits 页面可以编辑和删除习惯。

## 设置

Settings 页面包含 16 个配置分区：

### Timer（计时器）
- 番茄钟时长（默认 25 分钟，范围 10–120 分钟）
- 每日番茄上限（默认 8 个）

### Auto-Start（自动开始）
- 完成番茄钟后是否自动开始休息
- 休息结束后是否自动开始下一个番茄钟

### Work Time（工作时间）
- 设定工作时段（如 9:00–18:00）
- 空闲提醒间隔

### Sleep Time（睡眠时间）
- 设定睡眠时段
- 睡眠时段内的应用强制关闭

### Over Rest（超时休息）
- 休息超时后的行为配置
- 超时阈值设置

### REST Enforce（休息强制）
- 休息状态下自动关闭或隐藏工作应用

### Work Apps（工作应用）
- 定义"工作应用"列表，用于休息和睡眠时段的屏蔽

### Expectations（预期）
- 每日预期工作时间
- 每日预期番茄数量

### Notifications（通知）
- 完成提醒
- 声音设置

### Early Warning（预警）
- 进度落后时的提醒阈值和方式

### Browser（浏览器）
- URL 黑名单和白名单配置
- 专注模式下的网站屏蔽规则

### Entertainment（娱乐）
- 娱乐网站列表管理
- 娱乐网站屏蔽策略

### Demo Mode（演示模式）
- 演示令牌管理
- 演示展示设置

### Principles（编码原则）
- 个人编码标准和偏好（供 AI 助手参考）

### Devices（设备）
- 查看和管理已连接的客户端设备

### API Keys（API 密钥）
- 管理 MCP、Skill 和外部集成的 API 密钥

## 多客户端使用

VibeFlow 支持四个客户端，共享同一个后端，实时同步状态。

### Web（主客户端）

功能最完整的客户端，提供所有管理和分析功能。
- 地址：`http://localhost:3000`（本地开发）
- 包含 Dashboard、任务/项目/目标管理、统计分析等全部功能

### 桌面端（macOS）

基于 Electron 的 macOS 应用，专注于**强制执行**：

- **窗口管理**：FOCUS 状态下限制切换到非工作应用
- **系统托盘**：在托盘显示当前状态、倒计时和番茄进度
- **进程守护**（Guardian）：确保 VibeFlow 桌面端持续运行
- **绕过检测**：检测用户试图绕过限制的行为
- **REST 强制**：休息时段自动关闭工作应用

### 浏览器扩展（Chrome）

名为 **VibeFlow Browser Sentinel**，功能包括：

- **网站屏蔽**：FOCUS 状态下屏蔽黑名单网站，显示专注页面
- **娱乐模式管理**：根据系统状态控制娱乐网站访问
- **LOCKED/OVER_REST 限制**：在受限状态下强制执行浏览限制
- **屏保页面**：被屏蔽时显示鼓励专注的页面
- **浏览活动追踪**：记录网站访问数据，供统计分析使用

### iOS 客户端

基于 Expo 的移动端应用，主要功能：

- **状态显示**：查看当前系统状态（IDLE / FOCUS / OVER_REST）
- **番茄钟状态**：查看当前番茄钟的倒计时和进度
- **任务列表**：查看今日任务和习惯
- **Screen Time 集成**：利用 iOS Family Controls API 屏蔽分心应用
- **习惯打卡**：支持习惯的查看和管理
- **设置**：服务器连接配置

> 注意：iOS 客户端目前以只读为主，写操作（如创建任务、开始番茄钟）正在开发中。

### 状态实时同步

所有客户端通过 WebSocket（Socket.io）实时同步状态。在任一客户端的操作（如完成番茄钟）会立即反映到其他所有已连接的客户端。

## AI 助手集成（MCP）

VibeFlow 提供 MCP（Model Context Protocol）接口，让 AI 助手（如 Claude Code）能够直接操作你的任务和专注系统。

### 可用资源（只读查询）

通过 `vibe://` URI 读取：

| 资源 | 说明 |
|------|------|
| `vibe://context/current` | 当前工作上下文（项目、任务、状态） |
| `vibe://tasks/today` | 今日任务列表（含 Top 3） |
| `vibe://projects/active` | 活跃项目列表 |
| `vibe://projects/all` | 所有项目 |
| `vibe://user/goals` | 用户目标 |
| `vibe://state/current` | 当前系统状态 |
| `vibe://pomodoro/current` | 当前番茄钟详情 |
| `vibe://history/pomodoros` | 最近 7 天番茄历史 |
| `vibe://analytics/productivity` | 生产力分析数据 |
| `vibe://blockers/active` | 当前活跃的阻碍 |
| `vibe://timeline/today` | 今日活动时间线 |

### 可用工具（执行操作）

通过 `flow_*` 命令执行操作：

**番茄钟控制**
- `flow_start_pomodoro` — 开始番茄钟
- `flow_start_taskless_pomodoro` — 开始无任务番茄钟
- `flow_switch_task` — 切换当前任务
- `flow_complete_current_task` — 完成当前任务

**任务管理**
- `flow_create_task_from_nl` — 自然语言创建任务
- `flow_get_task` — 获取任务详情
- `flow_update_task` — 更新任务
- `flow_delete_task` — 删除任务
- `flow_complete_task` — 完成任务
- `flow_add_subtask` — 添加子任务
- `flow_move_task` — 移动任务到其他项目
- `flow_set_plan_date` — 设置计划日期
- `flow_get_backlog_tasks` — 获取待办任务
- `flow_get_overdue_tasks` — 获取过期任务
- `flow_batch_update_tasks` — 批量更新任务
- `flow_quick_create_inbox_task` — 快速创建收件箱任务

**项目管理**
- `flow_create_project` — 创建项目
- `flow_create_project_from_template` — 从模板创建项目

**分析与报告**
- `flow_get_task_context` — 获取任务上下文
- `flow_analyze_task_dependencies` — 分析任务依赖
- `flow_generate_daily_summary` — 生成每日总结
- `flow_report_blocker` — 报告阻碍

### 使用示例

在 Claude Code 中使用 VibeFlow Skill：

```
/vibeflow-tasks          # 查看今日任务
/vibeflow-focus          # 开始/结束专注
/vibeflow-projects       # 管理项目
/vibeflow-analytics      # 查看生产力数据
```

### 认证

- 开发模式：使用 `dev_<email>` 格式的 token
- 生产模式：在设置 > API Keys 中生成和管理 API 密钥

## 常见问题

### Q: 中止（Abort）和完成（Complete）番茄钟有什么区别？

**完成**：倒计时结束后的正常完成，计入当日番茄数，记录最后完成时间（可能触发 OVER_REST）。

**中止**：主动放弃本次番茄钟，不计入完成数，不会触发 OVER_REST。适用于被紧急事情打断的情况。

### Q: 什么是 OVER_REST？怎么退出？

当你完成一个番茄钟后休息太久，系统会自动进入 OVER_REST 状态，提醒你回来工作。退出方式有三种：
1. 直接开始一个新的番茄钟
2. 等待 10 分钟冷却期后点击「返回空闲」（每天最多 3 次）
3. 等到工作时间结束，系统自动退出

### Q: 如何让任务出现在"今日任务"中？

编辑任务，将「计划日期」（planDate）设为今天。只有 planDate 为今天的任务会出现在 Today 视图和 Dashboard 的任务列表中。

### Q: 番茄钟时长可以自定义吗？

可以。进入 Settings > Timer，可以调整单次番茄钟时长（10–120 分钟）和每日上限。

### Q: 多个客户端的数据会同步吗？

会。所有客户端共享同一个后端，通过 WebSocket 实时同步。在 Web 端完成番茄钟，桌面端和 iOS 端会立即更新状态。

### Q: 浏览器扩展会屏蔽哪些网站？

取决于你的配置。进入 Settings > Browser 设置黑名单/白名单，Settings > Entertainment 管理娱乐网站列表。在 FOCUS 状态下，黑名单中的网站会被屏蔽并显示专注页面。

### Q: iOS 端可以创建任务或开始番茄钟吗？

iOS 端目前以只读为主，写操作功能正在开发中。你可以在 iOS 端查看状态、任务列表和习惯，但创建和修改操作需要在 Web 端或通过 AI 助手完成。

### Q: 如何通过 AI 助手（如 Claude Code）操作 VibeFlow？
