# VibeFlow Design Context

## Users

**Primary audience**: 远程/独立工作者、知识工作者（程序员、设计师、作家）、创业者/自由职业者。

**Usage context**: 全天使用。白天在家办公桌前用 Web/Desktop 长时间专注工作，通勤/碎片时间用 iOS 快速查看状态和管理任务，Browser Extension 在浏览器中始终守护专注状态。

**Job to be done**: 用户有预期状态（工作、休息、睡眠、目标达成），现实往往偏离，系统持续校正帮用户回归预期。不是"管理任务"，而是"帮用户心想事成"。

**Emotional goal**: 用户应该感到被理解和支持，而不是被监控和管教。系统是温暖的陪伴者，不是冰冷的工具。

## Brand Personality

**3 words**: 温暖、鼓励、陪伴

**Voice**: 像一个贴心的教练——推你但不逼你，提醒你但不唠叨，为你的每一步进展真心高兴。

**Emotional goals**:
- 在专注时：安静守护，不打扰
- 在休息时：温和提醒，不催促
- 在完成时：真诚庆祝，不夸张
- 在偏离时：轻声引导，不责备

## Aesthetic Direction

**Visual tone**: 温暖的极简主义（Warm Minimalism）。干净但不冰冷，简约但不无聊。像一个精心布置的书房——木质暖调、自然光、有序但有温度。

**Theme**: Light 为主 + Dark 可选。白天工作场景光线充足，light 模式更舒适；夜间使用提供 dark 模式。

**References**:
- **Things 3**: Apple 原生感、精致细节、温暖的 light 主题、交互有质感
- **Forest**: 专注类应用的沉浸感、成长隐喻、完成后的满足感
- **Centered**: 极简专注界面、呼吸感的布局

**Anti-references** (绝对不要):
- 赛博朋克/霓虹风格
- 大量渐变和发光效果
- 密集的 dashboard 信息过载
- 游戏化过度（积分、排行榜、成就系统）
- 冰冷的科技感（纯黑+蓝色高光）

**Color direction**:
- Primary: 温暖的蓝绿色调（teal/warm blue），而非冰冷的蓝色
- Neutrals: 暖灰色，带有微妙的暖色倾向（不是 Zinc 冷灰）
- Accent: 自然的绿色调表示成功/完成（如植物生长的绿）
- Warning: 温暖的琥珀色
- Error: 柔和的红色，不刺眼

## Design Principles

1. **安静守护（Quiet Presence）**: 界面在你不需要时几乎消失，在你需要时温和出现。减少视觉噪音，增加意义密度。

2. **自然节奏（Natural Rhythm）**: 专注→休息→专注的节奏应该通过视觉传达——不同状态有不同的视觉氛围，切换自然流畅，像呼吸一样。

3. **温暖精致（Warm Craft）**: 每个细节都经过打磨。圆角、阴影、间距、动效都体现"有人在乎"。不是冰冷的效率工具，而是温暖的伙伴。

4. **一致但适应（Consistent yet Adaptive）**: 4 个客户端共享同一设计语言，但各自适应平台特性。Web 有键盘快捷键，iOS 有手势，Extension 有轻量弹出，Desktop 有 tray。

5. **内容优先（Content First）**: 用户关心的是任务和进度，不是界面本身。UI 服务于内容，让任务、时间、进度成为视觉焦点。

## Cross-Platform Design Tokens (Target)

所有 4 个客户端应从同一 token 源生成各自格式：
- Web: CSS Custom Properties + Tailwind config
- iOS: React Native theme object
- Extension: CSS Custom Properties
- Desktop: 继承 Web

Token 类别：Color, Typography, Spacing, Border Radius, Shadow, Motion, Icon
