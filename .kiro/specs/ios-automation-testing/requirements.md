# iOS 自动化测试

## 问题背景

当前 iOS 开发验证流程：
1. 改代码
2. `dev.sh ios --build --sim` 编译部署（30-60s）
3. 手动在模拟器上点击操作
4. 肉眼观察结果是否符合预期
5. 重复...

**痛点**：
- 每次验证需要 2-5 分钟人工操作
- 容易漏测边缘 case
- 回归测试靠记忆，改 A 功能可能悄悄破坏 B
- 跨端同步问题难以手动复现（需要同时操作 Web + iOS）

## 目标

用自动化测试替代重复的手动操作，实现：
- **改代码 → 跑命令 → 看结果**，无需手动点击
- 关键流程有回归保护，改坏了立刻知道
- 跨端同步场景可自动验证

## 需要覆盖的测试场景

### P0 — 核心流程（必须自动化）

1. **登录/注册**
   - 输入账号密码 → 登录成功 → 进入主页
   - 错误密码 → 显示错误提示
   - DEV 快速填入 → 自动填充测试账号

2. **任务管理**
   - 创建任务 → 列表中出现
   - 完成任务 → 状态变更
   - 删除任务 → 从列表消失

3. **番茄钟**
   - 启动番茄钟 → 倒计时显示
   - 完成番茄钟 → 状态切换到 REST
   - 中止番茄钟 → 返回 PLANNING

4. **跨端同步**
   - Web 创建任务 → iOS 自动刷新显示
   - iOS 启动番茄钟 → Web 状态同步

### P1 — 重要功能

5. **设置页面**
   - 修改每日番茄钟目标
   - 切换服务器地址

6. **Screen Time 集成**
   - 选择分心应用
   - 番茄钟期间验证屏蔽生效

7. **离线模式**
   - 断网操作 → 本地缓存
   - 恢复网络 → 自动同步

### P2 — 边缘场景

8. **错误处理**
   - 网络超时提示
   - 服务器错误提示

9. **状态恢复**
   - 杀掉 App 重开 → 状态保持
   - 切后台再回来 → 正常工作

## 技术选型

### 推荐方案：Maestro

**理由**：
- YAML 写测试，学习成本极低
- **支持模拟器和真机**（真机需 `idb-companion`）
- 不需要修改 App 代码
- 社区活跃，Expo 项目广泛使用

**设备支持**：

| 设备 | 支持 | 适用场景 |
|------|------|----------|
| iOS 模拟器 | ✅ 开箱即用 | 日常开发，快速迭代 |
| iOS 真机 | ✅ 需 idb | Screen Time、推送等真机特有功能 |
| Android | ✅ | 未来跨平台扩展 |

**对比其他方案**：

| 方案 | 上手难度 | 稳定性 | 速度 | 适用场景 |
|------|----------|--------|------|----------|
| **Maestro** | ⭐ 极简 | ⭐⭐⭐ | ⭐⭐⭐ | E2E 流程验证 |
| Detox | ⭐⭐⭐ 复杂 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 大型项目、CI 集成 |
| Appium | ⭐⭐⭐⭐ 很复杂 | ⭐⭐ | ⭐ | 跨平台、真机农场 |
| RNTL | ⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 组件单元测试 |

### 分层测试策略

```
┌─────────────────────────────────────┐
│  E2E (Maestro)                      │  ← 验证完整用户流程
│  - 登录 → 创建任务 → 启动番茄钟     │
│  - 跑一次 2-3 分钟                  │
├─────────────────────────────────────┤
│  组件测试 (RNTL)                    │  ← 验证单个组件交互
│  - LoginScreen 表单验证             │
│  - TaskCard 点击行为                │
│  - 跑一次 10 秒                     │
├─────────────────────────────────────┤
│  单元测试 (Jest)                    │  ← 验证纯逻辑
│  - Store 状态变更                   │
│  - Service 数据处理                 │
│  - 跑一次 2 秒                      │
└─────────────────────────────────────┘
```

## 预期收益

| 场景 | 当前耗时 | 自动化后 |
|------|----------|----------|
| 验证登录流程 | 3 分钟手动 | 30 秒命令 |
| 验证任务 CRUD | 5 分钟手动 | 1 分钟命令 |
| 跨端同步验证 | 10 分钟（开两个端） | 2 分钟命令 |
| 回归测试全流程 | 30 分钟手动 | 5 分钟命令 |

## 使用方式（目标体验）

```bash
# 快速验证登录（改了 LoginScreen 后跑）
./scripts/dev.sh ios:test login

# 验证任务流程
./scripts/dev.sh ios:test tasks

# 完整回归（提交前跑）
./scripts/dev.sh ios:test all

# 跨端同步测试（自动启动 Web + iOS）
./scripts/dev.sh ios:test sync
```

## 非目标

- 不追求 100% 覆盖率，优先覆盖手动测试最痛的场景
- 不做性能测试（有专门的 profiling 工具）
- 不做视觉回归测试（UI 改动频繁，维护成本高）

## 验收标准

1. P0 场景全部有自动化测试覆盖
2. 本地开发可一键运行测试
3. 测试失败有清晰的错误信息和截图
4. README 有完整的使用说明
